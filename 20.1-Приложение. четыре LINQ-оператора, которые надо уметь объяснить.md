Эти четыре оператора — высокочастотные на собесе, и спрашивают не реализацию, а **семантику**: что делает, в какой форме результат, когда брать. Модель выполнения (deferred / streaming / материализация) — в главе «LINQ to Objects»; здесь только контракты.

---

### `SelectMany` — расплющивание вложенности

**Что делает.** Проецирует каждый элемент в последовательность и **склеивает** все эти последовательности в одну плоскую. Контраст с `Select`: `Select(o => o.Items)` даёт `IEnumerable<IEnumerable<Item>>` (вложенность), `SelectMany(o => o.Items)` — `IEnumerable<Item>` (плоско).

![[Pasted image 20260625160000.png]]

```csharp
List<Order> orders = ...;  // у каждого Order есть List<Item> Items

IEnumerable<List<Item>> nested = orders.Select(o => o.Items);   // вложенно
IEnumerable<Item>       all    = orders.SelectMany(o => o.Items); // плоско

// overload с result-селектором: пара "родитель + ребёнок"
var pairs = orders.SelectMany(o => o.Items, (o, i) => new { o.Id, ItemName = i.Name });

// query-syntax: два from подряд компилируются в SelectMany
var all2 = from o in orders from i in o.Items select i;
```

**Типичный вопрос / грабли.** «Есть `List<Order>`, у каждого `Items` — собери все items одной последовательностью» — без `SelectMany` не решить чисто. Грабли: взять `Select` и получить `IEnumerable<IEnumerable<…>>`, потом удивляться вложенности; помнить, что `SelectMany` расплющивает **один** уровень (для более глубокой вложенности — вызвать ещё раз).

**Мини-ответ:** `SelectMany` проецирует каждый элемент в последовательность и склеивает их в одну плоскую; `Select` той же лямбдой дал бы последовательность последовательностей. В query-синтаксисе это два `from` подряд.

### `GroupBy` — группировка с ключом

**Что делает.** Группирует элементы по ключу. Форма результата — `IEnumerable<IGrouping<TKey, TElement>>`, где **`IGrouping<TKey, TElement> : IEnumerable<TElement>`**: каждая группа это и есть последовательность своих элементов, плюс свойство `.Key`. Это то место, где спотыкаются: группа — не пара «ключ-список», а _перечислимая последовательность_ с приклеенным ключом.

```csharp
// форма: IEnumerable<IGrouping<int, Order>>
foreach (var g in orders.GroupBy(o => o.CustomerId))
{
    int     customerId = g.Key;               // ключ группы
    decimal total      = g.Sum(o => o.Amount); // g — последовательность Order
}

// overload с result-селектором: сразу проекция на группу, без промежуточного IGrouping
var totals = orders.GroupBy(
    o => o.CustomerId,
    (key, items) => new { CustomerId = key, Total = items.Sum(o => o.Amount) });

// query-syntax
var q = from o in orders group o by o.CustomerId into g
        select new { g.Key, Total = g.Sum(o => o.Amount) };
```

**Типичный вопрос / грабли.** «Сгруппируй заказы по клиенту и посчитай сумму на каждого». Грабли: не понимать, что `g` — это последовательность (а `g.Key` — ключ); путать с `ToLookup`, который делает почти то же, но **немедленно** и возвращает готовый `ILookup<TKey, TElement>` для повторных обращений по ключу. `GroupBy` — отложенный и буферизующий (см. основную главу).

**Мини-ответ:** `GroupBy` возвращает последовательность групп `IGrouping<TKey, TElement>`; группа — это перечислимая последовательность элементов плюс `.Key`. Если нужна сразу проекция на группу — `(key, items) => …`; если нужна материализованная карта для повторных lookup'ов — `ToLookup`.

### `Aggregate` — свёртка (fold)

**Что делает.** Прогоняет аккумулятор по последовательности — обобщённый left-fold. Это та модель, частными случаями которой являются `Sum`/`Count`/`Min`/`Max`. Берут, когда готового хелпера нет. Выполняется **немедленно**. Три перегрузки:

```csharp
// 1) без seed: первый элемент — начальное значение.
//    НА ПУСТОЙ последовательности → InvalidOperationException
int sum = numbers.Aggregate((acc, x) => acc + x);

// 2) с seed: на пустой вернёт сам seed. Тип аккумулятора может отличаться от элемента
int product = numbers.Aggregate(1, (acc, x) => acc * x);

// 3) с seed и result-селектором (финальное преобразование аккумулятора)
string csv = names.Aggregate(
    new StringBuilder(),
    (sb, n) => sb.Append(n).Append(','),
    sb => sb.ToString().TrimEnd(','));
```

**Типичный вопрос / грабли.** «Сделай свёртку без готового агрегата» (произведение, конкатенация со状ом). Грабли: перегрузка **без seed бросает на пустой** последовательности — а с seed возвращает seed; и читаемость: если есть `Sum`/`Count`/`Max`, бери их, а не `Aggregate` ради «понтов».

**Мини-ответ:** `Aggregate` — обобщённый fold: seed плюс функция-аккумулятор, опционально финальный селектор. `Sum`/`Count` — его специализации; без seed он падает на пустой последовательности, поэтому для пустого-возможного источника нужен seed.

### `Join` / `GroupJoin` — соединение по ключам

**Что делает.** `Join` — **inner equi-join** двух последовательностей по равенству ключей, в памяти: возвращает плоскую последовательность пар, у которых ключи совпали. Только равенство ключей (не диапазоны), только совпавшие пары (несовпавшие outer-элементы выпадают).

![[Pasted image 20260625160022.png]]

```csharp
// inner equi-join
var rows = orders.Join(
    customers,
    o => o.CustomerId,            // ключ outer
    c => c.Id,                    // ключ inner
    (o, c) => new { o.Id, c.Name });

// GroupJoin: для каждого outer — ГРУППА подходящих inner (как left-join «в группу»)
var perCustomer = customers.GroupJoin(
    orders,
    c => c.Id,
    o => o.CustomerId,
    (c, cOrders) => new { c.Name, Count = cOrders.Count() });

// query-syntax
var q = from o in orders
        join c in customers on o.CustomerId equals c.Id
        select new { o.Id, c.Name };
```

**Типичный вопрос / грабли.** «Соедини заказы с клиентами по `CustomerId`». Грабли: думать, что `Join` делает outer-join по умолчанию — нет, только **inner** (несовпавшие outer выпадают); ждать range-условий (`<`, `BETWEEN`) — `Join` только по равенству. **LEFT JOIN** в LINQ собирают шаблоном `GroupJoin` + `SelectMany` + `DefaultIfEmpty`.

**Мини-ответ:** `Join` — inner equi-join по равенству ключей: выдаёт плоские пары совпавших элементов, несовпавшие outer выпадают. `GroupJoin` даёт для каждого outer группу подходящих inner — это основа левого внешнего соединения через `SelectMany` + `DefaultIfEmpty`.
